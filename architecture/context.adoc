= Context & Scope

This section defines the context and scope of the Bandcheck application architecture. It outlines the boundaries of the system, its interactions with external entities, and the key components that make up the architecture.

== Business Context

[mermaid]
....
graph LR
    U[Users] --> A[Bandcheck]
    A --> IdP[Identity Provider]
    A --> MA[Metal Archives]
    B[Band] --> A
    A --> L[Large Language Model]
....

Users::
    People that are either anonymous visitors or registered users of the Bandcheck platform. They interact with the system to discover music, read reviews, and contribute content warnings.

Bandcheck::
    The core application that provides the platform for music discovery and content warnings. It handles user interactions, data management, and content delivery.

Identity Provider::
    An external identity provider used for user authentication and authorization. It manages user credentials and provides secure access to the Bandcheck platform.

Band::
    Musical artists or groups that are the subjects of reviews and content warnings within the Bandcheck platform.

Large Language Model::
    Provides summaries on a large numbers of reviews for a band, generating an overall content warning summary to assist users in making informed decisions.

Metal Archives::
    An encyclopedia of metal bands. An import possibility from Metal Archives is being considered to populate the Bandcheck database with initial band data.

== Technical Context

[mermaid]
....
graph TB
    U[Users] -- HTTPS --> CF[Cloudflare<br/>DNS + CDN]
    CF -- HTTPS --> BC[Bandcheck Platform<br/>Frontend + Backend]
    BC -- storage/retrieval --> DB[(SQLite Database<br/>PersistentVolume)]
    BC -- OpenID Connect --> Auth0[Auth0<br/>Identity Provider]
    BC -- bot verification --> Turnstile[Cloudflare Turnstile<br/>Bot Protection]
    BC -- data import --> MA[Metal Archives]
    BC -- review summarization --> LLM[Large Language Model]
....

Users::
    Interact with the Bandcheck application through web browsers or mobile devices. They can browse content, submit reviews, and manage their profiles. All traffic is routed through Cloudflare for DNS resolution and CDN caching.

Cloudflare::
    Provides DNS management and CDN services. Handles traffic routing and SSL/TLS termination. Also provides Turnstile bot protection service.

Bandcheck Platform::
    The main application consisting of frontend (React SPA) and backend (Bun API) running on Kubernetes. Processes user requests, manages business logic, and interfaces with other system components.

SQLite Database::
    A persistent storage system on Kubernetes PersistentVolume that holds user data, reviews, content warnings, and band information. Embedded database without separate server process.

Auth0 (Identity Provider)::
    Provides authentication services via OAuth 2.0 / OpenID Connect, allowing users to securely log in using social providers (GitHub, Google, etc.) and access their accounts.

Cloudflare Turnstile::
    Bot protection service that verifies form submissions are from real users, not automated bots. Privacy-friendly alternative to traditional CAPTCHAs.

Metal Archives::
    An external data source that may be used to import band information into the Bandcheck database (future feature).

Large Language Model::
    A service that processes and summarizes user reviews to generate content warnings, enhancing the user experience by providing concise information (future feature).
