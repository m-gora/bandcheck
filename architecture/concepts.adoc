= Crosscutting Concepts

This section describes concepts and patterns that are relevant across multiple parts of the system.

== Security

=== Authentication & Authorization

**OAuth 2.0 / OpenID Connect:**

* All authentication handled via Auth0
* JWT tokens for API authorization
* Social login providers (GitHub, Google, etc.)
* No password storage in our system

**Token Management:**

* Access tokens stored in `localStorage` (frontend)
* Refresh tokens for session persistence
* Token validation on every protected API call
* Automatic token refresh via Auth0 SDK

**Authorization Flow:**

1. User initiates login
2. Redirect to Auth0 hosted login page
3. User authenticates with social provider
4. Auth0 issues JWT access token
5. Frontend stores token in `localStorage`
6. Token included in `Authorization: Bearer` header
7. Backend validates JWT signature and claims
8. Extract user identity from token claims

=== Bot Protection

**Cloudflare Turnstile:**

* Privacy-friendly CAPTCHA alternative
* Invisible to users in most cases
* Server-side verification required
* Protects form submissions (artist submit, reviews)

**Implementation:**

* Frontend: React Turnstile component generates token
* Backend: Verifies token with Cloudflare API before processing
* Test keys available for development
* Free tier with unlimited requests

=== CORS Configuration

**Backend CORS Settings:**

* Configured via `ALLOWED_ORIGINS` environment variable
* Comma-separated list of allowed domains
* Credentials enabled for Auth0 cookies
* Preflight requests handled automatically

**Example:**

[source,bash]
----
ALLOWED_ORIGINS=https://bandcheck.example.com,https://dev.bandcheck.example.com
----

=== Data Protection

**Secrets Management:**

* Kubernetes Secrets for sensitive configuration
* Environment variables never in source code
* Separate secrets per environment (dev/prod)
* ArgoCD manages secret deployment

**Encryption:**

* HTTPS/TLS for all communication

== Error Handling

=== Frontend Error Handling

**API Errors:**

* Caught by API service layer
* Translated to user-friendly messages
* Displayed via UI notifications (toast/snackbar)
* Failed requests can be retried

**Common Error Scenarios:**

* 401 Unauthorized → Prompt for login
* 403 Forbidden → "Permission denied" message
* 404 Not Found → "Resource not found"
* 500 Server Error → "Something went wrong, try again"

**Loading States:**

* Skeleton loaders during data fetch
* Disabled buttons during submission
* Clear indication of progress

=== Backend Error Handling

**Error Response Format:**

[source,json]
----
{
  "error": {
    "message": "Human-readable error message",
    "code": "ERROR_CODE",
    "status": 400,
    "details": {} // Optional additional info
  }
}
----

**Error Categories:**

* **Validation Errors (400):** Invalid input data
* **Authentication Errors (401):** Missing/invalid JWT
* **Authorization Errors (403):** Insufficient permissions
* **Not Found Errors (404):** Resource doesn't exist
* **Server Errors (500):** Unexpected failures

**Error Logging:**

* All errors logged with context
* No sensitive data in error messages
* Stack traces in development only
* Structured logging for parsing

== Logging

=== Log Levels

* **ERROR:** Application errors, failures
* **WARN:** Warnings, deprecated usage
* **INFO:** Application lifecycle, key events
* **DEBUG:** Detailed debugging (dev only)

=== Structured Logging

**Format:**

[source,json]
----
{
  "timestamp": "2026-02-07T12:34:56Z",
  "level": "INFO",
  "message": "Band created",
  "context": {
    "bandId": "123",
    "userId": "auth0|456",
    "ip": "192.0.2.1"
  }
}
----

**Best Practices:**

* Include correlation IDs for request tracing
* No PII in logs (use user IDs, not emails)
* Consistent field names across services
* Timestamp in ISO 8601 format

== Development Workflow

=== Local Development

**Prerequisites:**

* Node.js or Bun runtime
* Git for version control
* Docker (optional, for testing containers)
* kubectl (optional, for testing K8s manifests)

**Setup:**

[source,bash]
----
# Frontend
cd frontend
npm install
npm run dev

# Backend
cd backend
bun install
bun dev
----

**Environment Variables:**

* Copy `.env.example` to `.env`
* Configure Auth0 credentials
* Set Turnstile keys (use test keys for dev)
